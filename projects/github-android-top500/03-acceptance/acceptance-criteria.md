## 功能验收标准

### 功能 1: 数据爬取与更新

#### 验收场景 1.1: 首次数据获取
- **Given**: 系统刚部署，数据库为空
- **When**: 系统启动或手动触发数据刷新
- **Then**: 成功从GitHub API获取500个Android项目数据
- **And**: 数据包含所有必需字段（名称、描述、star数等）
- **And**: 项目按star数量降序排列

#### 验收场景 1.2: 定时自动更新
- **Given**: 系统正常运行中
- **When**: 到达定时更新时间（每24小时）
- **Then**: 自动触发数据更新
- **And**: 更新成功后，lastUpdateTime更新
- **And**: 前端页面显示最新数据

#### 验收场景 1.3: 手动触发更新
- **Given**: 用户在页面上操作
- **When**: 点击"刷新数据"按钮
- **Then**: 后端开始数据更新任务
- **And**: 前端显示"正在更新..."状态
- **And**: 更新完成后自动刷新列表

#### 验收场景 1.4: API速率限制处理
- **Given**: GitHub API请求接近速率限制
- **When**: 继续发送请求
- **Then**: 系统暂停请求，等待限制重置
- **And**: 显示友好的等待提示
- **And**: 限制重置后继续获取数据

#### 验收场景 1.5: 网络错误处理
- **Given**: 网络连接不稳定
- **When**: API请求失败
- **Then**: 系统自动重试（最多3次）
- **And**: 记录错误日志
- **And**: 显示错误提示，不影响已缓存数据的展示

---

### 功能 2: 项目列表展示

#### 验收场景 2.1: 默认列表加载
- **Given**: 用户访问网站首页
- **When**: 页面加载完成
- **Then**: 显示项目列表表格
- **And**: 默认每页显示20条记录
- **And**: 默认按star数量降序排列
- **And**: 显示分页控件

#### 验收场景 2.2: 分页功能
- **Given**: 项目总数超过20条
- **When**: 用户点击"下一页"或页码
- **Then**: 显示对应页面的数据
- **And**: 页面切换时间 < 1秒
- **And**: URL参数同步更新（?page=2）

#### 验收场景 2.3: 表格列显示
- **Given**: 列表页面已加载
- **When**: 查看表格内容
- **Then**: 显示以下列：排名、项目名称、描述、Stars、Forks、语言、更新时间
- **And**: 排名从1开始，与star排序一致
- **And**: 数字格式化显示（如：12,345）

#### 验收场景 2.4: 点击项目名称
- **Given**: 表格显示项目列表
- **When**: 用户点击项目名称
- **Then**: 在新标签页打开GitHub项目页面

---

### 功能 3: 搜索与筛选

#### 验收场景 3.1: 关键词搜索
- **Given**: 列表页面已加载
- **When**: 用户在搜索框输入关键词（如"retrofit"）
- **Then**: 实时过滤显示匹配的项目（名称或描述包含关键词）
- **And**: 搜索响应时间 < 500ms
- **And**: 搜索结果数量更新

#### 验收场景 3.2: 编程语言筛选
- **Given**: 列表页面已加载
- **When**: 用户选择语言筛选"Java"
- **Then**: 只显示language字段为"Java"的项目
- **And**: 筛选可与其他条件组合

#### 验收场景 3.3: Star数量范围筛选
- **Given**: 列表页面已加载
- **When**: 用户选择star范围"10,000+"
- **Then**: 只显示stars >= 10000的项目
- **And**: 筛选结果按当前排序规则排列

#### 验收场景 3.4: 更新时间范围筛选
- **Given**: 列表页面已加载
- **When**: 用户选择"最近一周更新"
- **Then**: 只显示pushedAt在最近7天内的项目

#### 验收场景 3.5: 清除筛选
- **Given**: 已应用筛选条件
- **When**: 用户点击"清除筛选"或"重置"
- **Then**: 移除所有筛选条件
- **And**: 显示完整列表

- **Then**: 切换为新的排序列

### 功能 5: Excel导出

#### 验收场景 5.1: 导出全部数据
- **Given**: 列表页面已加载
- **When**: 用户点击"导出Excel"按钮（无筛选条件）
- **Then**: 下载包含全部500条数据的Excel文件
- **And**: 文件名格式为 github-android-top500-YYYYMMDD-HHmmss.xlsx
- **And**: Excel包含所有字段列

#### 验收场景 5.2: 导出筛选数据
- **Given**: 已应用筛选条件（如语言=Java）
- **When**: 用户点击"导出Excel"
- **Then**: 只导出当前筛选结果
- **And**: 导出文件数量与页面显示一致

#### 验收场景 5.3: Excel格式验证
- **Given**: 已导出Excel文件
- **When**: 用Excel打开文件
- **Then**: 表头正确显示
- **And**: 数据格式正确（日期、数字）
- **And**: 中文无乱码
- **And**: 自动列宽适配内容

#### 验收场景 5.4: 大数据导出性能
- **Given**: 导出500条数据
- **When**: 执行导出操作
- **Then**: 导出时间 < 5秒
- **And**: 导出过程中UI不卡顿

## 验收测试用例

### 测试用例 1: 完整用户流程
```
1. 访问首页
2. 确认列表正常加载（20条数据）
3. 点击第2页，确认分页正常
4. 输入搜索关键词"glide"，确认搜索结果正确
5. 选择语言"Kotlin"，确认筛选生效
6. 点击Stars列头，确认排序切换
7. 点击导出按钮，确认Excel下载成功
8. 打开Excel确认数据正确
```

### 测试用例 2: 数据刷新流程
```
1. 查看当前更新时间
2. 点击刷新按钮
3. 确认显示"更新中"状态
4. 等待更新完成
5. 确认更新时间已变更
6. 确认列表数据刷新
```

### 测试用例 3: 边界条件测试
```
1. 搜索不存在关键词 → 显示空结果
2. 导航到超出一半的页码 → 显示对应页
3. 同时应用多个筛选 → 组合生效
4. 快速连续点击刷新 → 正确处理（防抖）
```
