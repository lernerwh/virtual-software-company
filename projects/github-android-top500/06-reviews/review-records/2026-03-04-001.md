### ⚠️ 警告项 (5项) - 需要修复

#### W-001: Repository初始化重复
- **位置**: `backend/src/index.ts:22`, `backend/src/repositories/repoRepository.ts:14`
- **描述**: `initTable()` 在 index.ts 和 Repository 构造后都调用了
- **风险**: 中 - 可能导致重复建表
- **建议**: 移除 repoRepository.initTable() 调用，统一在 database/init.ts 初始化

#### W-002: 类型断言过多
- **位置**: `backend/src/services/crawlerService.ts:127-155`
- **描述**: transformRepo 方法中使用大量类型断言 `as`
- **风险**: 低 - 运行时可能类型不匹配
- **建议**: 定义 GitHub API 响应类型，避免使用 `Record<string, unknown>`

#### W-003: 搜索输入未清理
- **位置**: `backend/src/repositories/repoRepository.ts:105`
- **描述**: 搜索关键词直接拼接到 LIKE 语句，虽然有参数化，但特殊字符可能导致问题
- **风险**: 低 - 已使用参数化查询
- **建议**: 添加搜索关键词清理函数，转义 % 和 _ 字符

#### W-004: 前端错误处理不完整
- **位置**: `frontend/src/App.tsx:54-56`
- **描述**: 错误处理只显示一次 message，没有清除机制
- **风险**: 低 - 用户体验
- **建议**: 使用 React Query 的 onError 回调或 useEffect 处理错误

#### W-005: 导出功能无进度提示
- **位置**: `frontend/src/components/ExportButton/ExportButton.tsx:27`
- **描述**: 大数据量导出时无进度反馈
- **风险**: 低 - 用户体验
- **建议**: 添加 loading 状态，或使用消息提示
